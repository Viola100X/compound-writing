# 发布复盘 Skill
> **触发词**：「帮我做发布复盘」「分析推特数据」「月度复盘」「内容复盘」

---

## 数据输入方式

用户可以通过以下方式提供数据（优先级从高到低）：

### 方式 A：Twitter Analytics CSV 导入（推荐）

用户从 Twitter 后台导出 CSV：`Settings → Analytics → Export`，文件名通常为 `tweet_activity_metrics_*.csv`。

```
用户：「月度复盘」+ 提供 CSV 文件路径
系统：自动解析 CSV，提取所有字段
```

CSV 标准字段映射：
| CSV 列名 | 用途 |
|----------|------|
| Tweet text | 推文内容 |
| time | 发布时间 |
| impressions | 曝光量 |
| engagements | 互动量 |
| likes | 点赞 |
| retweets | 转发 |
| replies | 回复 |
| bookmarks | 收藏 |
| url clicks | 链接点击 |
| profile clicks | 主页点击 |

### 方式 B：手动输入

- 逐条粘贴推文内容 + 曝光/点赞/书签/评论数据
- 截图识别

### 方式 C：Apify 拉取（Web 版）

Web 版可复用 twitter-analyzer 的 Apify 额度自动拉取。

## 分析框架：三个维度

---

### 维度一：整体数据分析

**1a — 核心数据一眼看**

输出格式：
```
| 指标 | 数值 | 解读 |
|------|------|------|
| 分析周期原创发推 | X条 | 约X条/周 |
| 回复数 | X条 | 是发推的X倍 |
| 平均曝光 | X | [解读] |
| 中位曝光 | X | [解读] |
| 最高单条曝光 | X | |
| 点赞中位数 | X | [解读] |
| 书签中位数 | X | [解读] |
```

**1b — 爆款情况盘点**

按曝光量从高到低列出 Top 5 内容，每条附：
- 内容摘要（前20字）
- 关键数据
- 一句话判断：「为什么这条跑出来了」

**1c — 发布频率 review**

用 ASCII 图直观展示原创/回复结构：
```
原创发推  ████████░░░░░░░░  X条 (X%)
回复/对话 ████████████████  X条 (X%)
```

判断：
- 发布频率是否稳定？
- 原创 vs 回复比例是否健康（建议原创占比 ≥ 60%）

**1d — 内容方向与商业目标一致性评估**

按话题类型分类，评估每类内容是否服务于账号的商业定位：

```
| 话题类型 | 条数 | 平均曝光 | 与商业目标关联度 |
|---------|------|---------|----------------|
| [类型1] | X条  | X       | ✅/⚠️/❌        |
| [类型2] | X条  | X       | ✅/⚠️/❌        |
```

结论：哪些内容在强化定位，哪些在消耗定位。

---

### 维度二：具体爆款拆解

针对本期 Top 3 爆款，逐条分析：

**2a — 为什么是优质内容**

从三个角度判断：
- 数据表现（曝光/互动是否显著高于均值）
- 内容质量（是否有独立观点、具体案例、独特视角）
- 时效性（是否借势热点）

**2b — 内容结构拆解**

对应系统中的结构类型（S1-S4 短文 / A-D 长文）：
- 这条用了什么结构？
- Hook 是怎么写的？
- 关键转折/爆点在哪里？
- 结尾怎么处理的？

**2c — 用户痛点分析**

这条内容戳中了读者的哪个层面：

| 痛点类型 | 说明 | 判断 |
|---------|------|------|
| 认知缺口 | 读者不知道这件事，知道后有「原来如此」感 | ✅/- |
| 身份认同 | 让读者觉得「说的就是我」 | ✅/- |
| 实用性 | 读完可以立刻用，收藏备用 | ✅/- |
| 社交货币 | 转发出去能让自己显得聪明/消息灵通 | ✅/- |
| 情绪共鸣 | 说出了读者憋在心里的话 | ✅/- |

---

### 维度三：后续内容规划

**3a — 基于数据的方向建议**

```
立刻停止：
❌ [基于低效内容数据，具体说明停止什么]

继续加强：
✅ [基于爆款规律，具体说明强化什么]

值得尝试：
→ [根据账号定位，提出1-2个新方向建议]
```

**3b — 选题公式提炼**

基于本期爆款，总结出「当前有效的选题公式」：

```
[公式]: [元素A] + [元素B] = 高效内容
示例选题：
1. [具体选题方向]
2. [具体选题方向]
3. [具体选题方向]
```

**3c — 下期行动清单**

```
- [ ] [具体可执行的行动1]
- [ ] [具体可执行的行动2]
- [ ] [每条推文写完问自己：这条只有我能发吗？]
```

## 输出规范

参考风格：直接、有数据、有具体内容引用、结论清晰。

- 每个维度单独成节，标题清楚
- 数据用表格展示，趋势用 ASCII 图
- 引用原文时用 > 块引用格式
- 结论句用 **加粗**，一眼可见
- 避免模糊表达（「很多」→「X条，占X%」）

## 可视化报告生成

JSON 生成后，自动生成自包含 HTML 报告：

1. 读取 `visualizer/monthly-review.html` 模板
2. 将 JSON 嵌入为 `window.__COMPOUND_REVIEW_DATA__ = {...}`
3. 输出到 `visualizer/reports/review-{handle}-{YYYY-MM}.html`
4. 提示用户打开查看

## 复盘报告保存位置

分析完成后，保存至以下路径。**若目录不存在请先用 `mkdir -p` 创建**（首次使用时这些目录都还没建立）：

```
04-发布复盘/YYYY-MM-月度复盘.md              ← 文字版分析
visualizer/reports/review-{handle}-{YYYY-MM}.html  ← 可视化版
visualizer/reports/review-{handle}-{YYYY-MM}.json  ← 原始 JSON
```

高效内容的结构和句式，同步更新进知识库：
```
02-knowledge-base/爆款素材/
```

## 学习闭环

发布后数据可以回喂到选题和文风 Skill，持续校准系统：

| 时间点 | 收集数据 | 分析目的 |
|-------|---------|---------|
| 24h | 初始表现 | 判断是否进入推荐池 |
| 7d | 稳定表现 | 评估实际传播效果 |
| 30d | 长尾表现 | 分析内容长期价值 |

通过对比「评估预测」和「实际表现」，持续校准评估模型的准确性。
