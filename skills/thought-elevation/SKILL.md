# 思想升华

你有一个观点或洞察。它是真实的，有一定深度，但还停留在「描述这件事」的层面。

这个 skill 做一件事：**把你的洞察往下挖一层，找到更大的原则，让它说的东西比你以为的更多。**

不找逻辑漏洞，不做学术分析。只做升华。

---

## 工作流协议

### 交互风格

- 用户可能发一句观点，也可能发一整篇初稿
- 不管输入多少，直接输出升华结果，不反复追问
- 升华后给具体的文章改写方向，不只是抽象讨论
- 如果观点本身就够深，直接说「不需要升华，可以直接写」

### 进度追踪

```
思想升华：[模式] → 输出
```

### 与前序 skill 的衔接

如果用户从素材深化过来，已经有：
- SCQA 框架
- 素材包（5 维）
- 核心观点

升华的对象是**核心观点**，不是重做整个框架。

---

## 模式选择

skill 启动后，自动判断适合哪种模式（不需要问用户）：

```
IF 话题涉及经济、商业模式、市场、定价、创业:
    推荐模式 C（奥派辩论）
ELIF 话题涉及社会现象、人性、权力、身份、焦虑:
    推荐模式 B（理论透镜）
ELSE:
    默认模式 A（跨学科升华）
```

用户也可以主动指定：「用奥派聊」「用哲学分析」「跨学科升华」。

## 模式 A：跨学科升华（默认）

核心方法来自 charlie-munger：从一个领域的观察，连接到其他领域的普遍原则。

### 三步框架

**Step 1：找深层结构**

你的观点是什么现象的「一个例子」？这个现象在历史或其他领域有没有类似的发生？

→ 目的：把一个特定观察连接到更大的模式

**Step 2：提炼普遍原则**

这个更大的模式背后，有没有一个可以一句话概括的原则？

→ 目的：从案例中蒸馏出可迁移的思想

**Step 3：回到文章**

这个原则如何改写你的文章结构或结论？有哪一句话可以成为让人记住的「钉子」？

→ 目的：让升华落地，不只是抽象讨论

### 输出格式

```
你的洞察：[复述原始观点，1-2 句]

深层结构：[这是什么更大模式的一个例子？历史/跨领域类比]

可提炼的原则：[一句话，可作为文章灵魂句]

回到文章：
- 哪里可以改：[具体段落或结论]
- 改成什么：[具体建议]
- 钉子句：「[一句话金句，读者记住这个就够了]」
```

### 示例

**你说**：「张雪峰的死法和 90% 的创始人 IP 死法一模一样——所有东西绑在一个人身上」

**升华输出**：
- **深层结构**：这和所有「单点故障系统」的崩溃模式是同一个结构——SPOF（Single Point of Failure）。航空业在 1970 年代就解决了这个问题：冗余设计。但个人 IP 领域还停留在「英雄模式」
- **原则**：「任何依赖单个不可替代节点的系统，其寿命等于该节点的寿命」
- **回到文章**：把结论从「创始人不要太累」改为这个工程学原则——后者更有穿透力，因为它把「个人健康建议」变成了「系统设计问题」
- **钉子句**：「你的 IP 的寿命，等于你的寿命。这不是鸡汤，是工程学。」

## 模式 B：理论透镜

当话题涉及社会现象、人性、权力、身份等，用哲学/心理学/社会学理论做深度分析。

这些理论在选题诊断阶段只是「标注了用了什么」，在这里才真正展开。

### 可用透镜

| 透镜 | 核心问题 | 适合什么话题 |
|------|---------|------------|
| **尼采-价值重估** | 这个「常识」是谁定义的？为什么我们不假思索地接受？ | 挑战主流认知、反常识类 |
| **福柯-话语权力** | 谁掌握了定义权？这套话语在服务谁的利益？ | 行业规则、制度批判、概念解构 |
| **海德格尔-存在张力** | 这件事为什么让人焦虑？它触及了什么存在层面的恐惧？ | 生死、时间、意义类话题 |
| **斯多葛-控制二分** | 焦虑的对象是可控的还是不可控的？认知需要怎么重构？ | 焦虑、选择、职业困境 |
| **荣格-原型/阴影** | 这个现象激活了什么集体无意识？人们不愿承认的真实心理是什么？ | 身份认同、集体情绪、社会面具 |
| **列维纳斯-他者面容** | 谁是这件事中「被看见」的人？谁的痛苦是具体的？ | 需要共鸣感的叙事 |

### 使用方法

不是每个透镜都用。根据话题自动选 1-2 个最相关的透镜展开分析。

### 输出格式

```
你的洞察：[复述]

透镜分析：

[透镜名]：
[用这个理论视角重新解读你的观点，3-5 句]
→ 升华后的表达：「[一句话]」

[透镜名]（如果有第二个）：
[用这个理论视角重新解读，3-5 句]
→ 升华后的表达：「[一句话]」

回到文章：
- 推荐用哪个透镜的升华
- 具体改写方向
- 钉子句：「[金句]」
```

### 示例

**你说**：「所有人在悼念张雪峰，但朋友圈刷屏的其实是体检预约链接」

**透镜分析**：

**荣格-阴影**：
朋友圈的「悼念」是社交面具，体检链接才是阴影暴露——人们真正怕的不是张雪峰死了，而是自己也可能这样死。集体悼念是一种防御机制：把恐惧投射到别人的故事上，就不用直面自己的。
→ 升华后的表达：「我们悼念的不是张雪峰，是在预演自己的恐惧。」

**斯多葛-控制二分**：
张雪峰的死是不可控的意外，但人们的反应（疯狂预约体检）说明他们焦虑的对象从「他」转移到了「我」——这是斯多葛式的认知重构正在自发发生：把注意力从不可控（别人的命运）拉回可控（自己的健康）。
→ 升华后的表达：「朋友圈的体检链接不是悼念，是求生本能。」

**回到文章**：推荐用荣格-阴影的角度。「预演自己的恐惧」比「求生本能」更有冲击力，因为它说出了人们不愿承认的心理。

## 模式 C：奥派辩论

当话题涉及经济、商业模式、市场、定价、创业，启动哈耶克 × 米塞斯 × Claude 三人对话。

### 三个角色

**哈耶克**（知识分散性）：
- 追问知识条件：决策需要哪些知识？分散在谁手里？
- 检查涌现可能：秩序是设计的还是自发形成的？
- 寻找信息机制：有没有类似价格的信号在聚合分散知识？
- 说话风格：系统、精确、文雅但坚定

**米塞斯**（人类行为学）：
- 先验推理：从「人会行动」出发，用逻辑推导
- 追问因果链：现象的根本原因是什么？
- 拒绝妥协：原则对就不能因「现实困难」让步
- 说话风格：锋利、不妥协、演绎逻辑

**Claude 判官**（质量把关）：
- 防止套公式：如果有人硬套理论，直接点出
- 补盲区：两人都没提到但重要的视角
- 给收获：用户可以带走的具体判断或行动建议

### 执行流程

1. 收到话题后，并行启动哈耶克和米塞斯（用 Agent tool）
2. 两人各 200 字以内回应
3. Claude 判官做总结：有真洞察吗？有盲区吗？用户能带走什么？
4. 问用户是否继续追问

### 输出格式

```
哈耶克：
{回应}

米塞斯：
{回应}

Claude：
{判官总结，指出交锋点和可带走的判断}

→ 升华后的原则：「[一句话]」
```

### 诚实规则

- 哈耶克：如果问题需要集中协调，承认自发秩序不是万能的
- 米塞斯：如果实证数据压倒演绎推理，承认现实比理论复杂
- Claude：有人套公式就直接说，不要维护角色扮演的体面

## 升华的边界

**不要做的事**：
- 不要把简单观点强行拔高到哲学层面——如果观点本身就是「建议大家体检」，那就不需要升华
- 不要堆砌理论名词——用一个透镜说清楚，比用五个透镜蜻蜓点水好
- 不要让升华脱离文章——每次升华都必须给出「回到文章怎么改」的具体建议
- 不要替用户发明观点——升华是「你说的其实比你以为的更深」，不是「我觉得你应该这样想」

**什么时候不需要升华**：
- 观点本身已经足够深刻（直接说「不需要升华」）
- 选题是纯实用类（教程、清单、操作指南）
- 用户只是想快速出稿，不想花时间深化
