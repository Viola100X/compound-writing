# 爆款内容结构选择器 Skill
> **触发词**：「帮我选结构」「这个用什么结构写」「我有个写作 idea」「写长还是写短」

**启动前先读 `.compound/session.md`**：
- 如果有「选题」段落 → 直接拿用，不要问用户"你想写什么"
- 如果没有 → 提示用户"要不要先说『帮我找选题』把主题先定下来？或者直接告诉我你想写什么，我帮你套结构"

触发后，Claude 做四件事：
1. **判断长短**：≤350 字短文 vs >350 字长文
2. **诊断类型**：短文 S1-S4 / 长文 A-D
3. **给出结构框架**：含亮点技巧
4. **列出素材清单**：写前必备

详细结构框架和真实案例见 `frameworks.md`

---

## 第一步：长短判断

| 判断问题 | 选短文 | 选长文 |
|---------|-------|-------|
| 观点能一句话概括？ | ✅ 是 | ❌ 有层次 |
| 需要几个案例支撑？ | 1 个以内 | 2 个以上 |
| 读者 30 秒能读完？ | ✅ 是 | ❌ 需深度阅读 |
| 目的是传播/共鸣？ | ✅ 是 | 偏权威/收藏 |

---

## 第二步：类型诊断

**短文（≤350 字）**：
- 你的核心是一条可清单化的经验 → **S1 清单型**（爆款率最高）
- 你要给一个现象命名 → **S2 命名现象型**
- 你要提出一个大胆声明 → **S3 反常识声明型**
- 你要给出一个反常识的警示 → **S4 警示忠告型**

**长文（>350 字）**：
- 「我发现大多数人理解错了 X」→ **A 型 认知颠覆型**
- 「我有一个方法/工具解决 X」→ **B 型 方法教程型**
- 「我经历了 X，学到了 Y」→ **C 型 故事叙事型**
- 「行业正在发生 X 变化」→ **D 型 趋势预判型**

详细框架见 `frameworks.md`

---

## 第三步：输出格式

```
## 诊断结果
长短判断：[短文 ≤350字 / 长文 >350字]
内容类型：[S1-S4 / A-D 类型名]
爆款逻辑：[为什么这个方向有潜力，1句]
传播预判：[谁会分享/收藏]

---

## 结构框架
[完整骨架：短文逐句结构；长文分段比例]

---

## 亮点设计
1. Hook：[如何让读者停下来]
2. 核心爆点：[短文=关键转折句；长文=神转折位置]
3. 结尾：[如何制造余韵或升华]

---

## 素材清单
写前必须准备：
- [ ] [素材1]
- [ ] [素材2]
- [ ] [素材3]
```

---

## 写前自检（先问自己 4 个问题）

1. 这个 idea 最近 7 天有热点事件吗？→ 有则加时效性 Hook
2. 我有真实实践/踩坑经历吗？→ 无则先补素材
3. 我能提供 2 个以上真实案例吗？→ 最少 2 个，否则说服力不足
4. 这个话题有争议性/矛盾点吗？→ 无争议则创造矛盾 Hook

4 个都 YES → 开始写 | 有 NO → 先补素材，不要硬写

---

## 完成后写入 session.md

诊断完成后，把结果追加/更新到 `.compound/session.md` 的「结构」段落：

```markdown
## 结构
- 长短：短文/长文
- 类型：S1-S4 / A-D
- 爆款逻辑：一句话
- 骨架：（保留完整骨架）
- 素材清单：（逐项）
```

并更新 frontmatter：`current_phase: structure`。

然后引导下一步（按中文触发词）：
- 素材齐 → "说『用我的文风写』直接出初稿"
- 缺素材 → "说『采访我』把素材补上"
- 想更深 → "说『升华一下』提升观点深度"
