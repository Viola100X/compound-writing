# All-in-one Inbox Skill

> **触发词**：「存一下」「记录」「inbox」「找关于X的素材」

---

## 这个 Skill 做什么

解决创作者最常见的问题：**灵感来了没地方记，记了又找不到。**

你可以在任何时候、任何渠道（Telegram、WhatsApp、微信、Claude Code 对话中），把想法、素材、链接发过来。系统自动存档，创作时一搜就能找到。

---

## 功能一：存档

### 触发方式

在对话中说「存一下」「记录这个」「inbox」，后面跟上你要存的内容。

### 支持的输入类型

| 输入类型 | 处理方式 |
|---------|---------|
| **文字消息** | 直接存档，AI 自动提取标签 |
| **语音消息** | OpenClaw 自动转文字（内置 Whisper），转写结果存档 |

### 存储格式

每条素材存为一个 Markdown 文件，保存到 `02-knowledge-base/_inbox/`

文件命名：`YYYY-MM-DD-HH-MM-{摘要}.md`

文件内容：
```markdown
---
date: 2026-03-10 14:30
source: telegram
type: text
tags: [AI, agent, 创业]
---

今天和一个做 AI Agent 的创始人聊，他说了一句很有意思的话：
"Agent 的问题不是不够聪明，是不够可信。用户不怕它做错事，怕的是不知道它在做什么事。"

这个观点可以写一篇关于 Agent 信任机制的文章。
```

### AI 自动处理

存档时 Claude 自动完成：
1. **提取标签**：从内容中识别 2-5 个关键词作为 tags
2. **生成摘要**：取前 10 个字作为文件名摘要
3. **判断类型**：text / voice / link
4. **确认存储**：输出「已存档：[文件名]，标签：[tags]」

## 功能二：搜索

### 触发方式

「找关于 X 的素材」「我之前存过一个关于 Y 的想法」「搜索 inbox」

### 搜索逻辑

1. 在 `02-knowledge-base/_inbox/` 目录中搜索
2. 匹配范围：文件内容 + tags + 文件名
3. 按时间倒序展示匹配结果

### 输出格式

```
## 搜索结果：关于「AI Agent」的素材（共 3 条）

### 1. 2026-03-10 — Agent 信任机制
> "Agent 的问题不是不够聪明，是不够可信..."
标签：AI, agent, 创业
→ 选题潜力：⭐⭐⭐⭐（有独特视角，可写认知差型）

### 2. 2026-03-08 — Agent 工具对比
> 测了 5 个 Agent 框架，LangGraph 和 CrewAI 差距很大...
标签：AI, agent, 工具
→ 选题潜力：⭐⭐⭐（可写方法论型，但需要补充数据）

---

要用哪条素材开始创作？选中后我帮你进 topic-engine 评估选题质量。
```

## 功能三：创作衔接

用户选中某条素材后，自动推荐下一步：

```
素材选中
    ↓
topic-engine（评估选题质量，五维打分）
    ↓
content-structure（选择最佳结构）
    ↓
开始创作
```

## 知识库目录结构

```
02-knowledge-base/
├── _inbox/              ← inbox-capture 存档位置
│   ├── 2026-03-10-14-30-agent信任机制.md
│   ├── 2026-03-10-09-15-推特运营想法.md
│   └── ...
├── 本专业信息/           ← 个人背景、产品信息
└── 爆款素材/             ← 按行业分类的参考素材
    ├── AI/
    ├── Web3/
    └── ...
```

`_inbox/` 是你的原始素材池。积累到一定量后，可以整理到 `爆款素材/` 的对应分类中。
