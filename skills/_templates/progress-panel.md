# 进度面板

每个阶段转换时展示此面板，让用户始终知道自己在哪。保持简洁，不用 emoji 或装饰线条。

## 格式

```
进度：Compound — {当前阶段名}

选题：{选题内容 / 待定}
结构：{结构类型 / 待定}
文风：{已初始化 / 通用风格}

[x] Personal OS    — {已完成 / 已跳过}
[x] 收集素材       — {素材数量 / 已跳过}
[>] 选题评估       — {评分/50 或"进行中"}
[ ] 选结构         — {结构类型 / 未开始}
[ ] 深化内容       — {轮次 / 未开始}
[ ] 出初稿         — {字数 / 未开始}
[ ] 标题/钩子      — {方案数 / 未开始}
[ ] 发布复盘       — {评分 / 未开始}
```

## 状态符号

- `[x]` — 已完成
- `[>]` — 当前进行中
- `[ ]` — 待完成
- `[-]` — 已跳过

## 示例

```
进度：Compound — 深化内容

选题：为什么我们会被张雪打动
结构：C 型故事叙事型
文风：通用风格（未初始化）

[-] Personal OS    — 跳过
[-] 收集素材       — 跳过
[x] 选题评估       — 37/50 A 级
[x] 选结构         — C 型故事叙事型
[>] 深化内容       — 理论透镜进行中
[ ] 出初稿
[ ] 标题/钩子
[ ] 发布复盘
```

## 阶段导航（面板后展示）

面板之后按 ask-format 给出选项。不要用"A) 继续 / B) 跳到 / C) 暂停"这种机械三连，而要根据当前的总编辑判断给出**具体的**三个方向（例如"A) 继续升华第 3 轮 / B) 直接进初稿 / C) 回去补素材"），并给出建议。
