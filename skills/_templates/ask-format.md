# 统一交互格式

每次向用户提问或引导下一步时，**必须**按以下 4 部分结构输出：

## 结构

1. **定位**（1-2 句）：当前在哪个阶段，刚完成了什么。
   假设用户已经 20 分钟没看这个窗口了。

2. **下一步**（1 句）：用大白话解释接下来要做什么决定。
   不用术语，不用实现细节。一个非技术创作者应该能看懂。

3. **建议**：始终给出一个推荐选项 + 理由。
   格式：`建议：选 [X]，因为 [一句话理由]`

4. **选项**：A/B/C 字母编号，附简短描述。
   需要时附上时间估算。

## 示例

```
我们刚完成了选题评估，你的选题「为什么我们会被张雪打动」
得了 37/50 分（A 级），值得写。

下一步是选择文章结构——长文还是短文，用什么骨架。

建议：选 C 型故事叙事型，因为你的核心是个人感悟 + 案例驱动，
这个结构最适合情感共鸣类内容。

A) 用 C 型故事叙事型 → 进入深化阶段
B) 换其他结构 → 我重新诊断
C) 跳过选结构 → 直接出初稿
```

## 禁止

- 不要只列选项不给建议
- 不要用技术术语（"调用 content-structure Skill 的 S1-S4 诊断"）
- 不要一次问多个问题（一次只问一件事）
- 不要没有上下文就开始问（先说清楚当前在哪）
