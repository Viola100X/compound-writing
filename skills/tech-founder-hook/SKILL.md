## 一、这个 Skill 解决什么问题

技术型创始人写内容有一个通病，容易写出平铺直叙的内容：技术观点——逻辑推演——结论

这个结构本身没错，但它前期调动多巴胺不足，缺乏让人看下去的动力。

读者在推特上的决策时间不超过三秒。他不会等你铺垫完再决定要不要继续读。

**这个 Skill 专门解决短推文开头的情绪钩子问题。**

---

## 二、情绪钩子的底层逻辑

情绪钩子不是标题党，不是制造悬念的技巧。

它的本质是：**唤醒读者沉睡的情绪记忆，让他觉得「终于有人懂我」或「然后呢？」**

你不是在设下圈套，而是在递给读者一把钥匙——开门之后，他发现里面真的有东西。

> 📌 前3秒决定用户停留，前3句决定最终去留。情绪钩子是这个黄金窗口的「转换器」。

**三大情绪软肋**，读者最容易被这三类内容触发停留：
- **共鸣**：「这说的不就是我」——被理解的需求
- **好奇**：「然后呢？为什么会这样」——认知探索欲
- **矛盾**：「这跟我想的不一样」——认知被挑战的反应

## 三、短推文的 5 种开头钩子

### 类型一：画面钩子
**定义**：用具象场景替代抽象情绪，激活读者的视觉记忆。

读者看到抽象描述「我当时很尴尬」，大脑不会产生画面。
读者看到具体场景，大脑会自动代入。

**对比**：
```
❌ 我当时特别尴尬
✅ 端着咖啡转身的瞬间，正好撞上领导的目光，
   而我的电脑屏幕上，还亮着没关的招聘网站
```

三个要素缺一不可：**时间节点 + 动作 + 让人意外的细节**

---

### 类型二：对立钩子
**定义**：呈现矛盾或价值冲突，制造认知张力。

**公式**：`所有人都认为 A` → `但我/但事实是 B`

**案例**：
```
所有人都劝我稳定点，可我才25岁，
为什么要活得像52岁？
```

**@tmel0211 的用法**：
```
虽然 $Molt 的币价已经趴在谷底了，
但 Agent 的注册量已经突破了260万
```
→ 币价（大家关注的）vs 注册量（反向增长）。两个数字一对比，张力就出来了。

---

### 类型三：认知钩子
**定义**：提出与常识相反的洞见，触发读者反思「为什么会这样」。

**公式**：`大家以为的 A` → `其实真正的情况是 B（更反直觉）`

**案例**：
```
我发现，那些从不抱怨的人，往往最先崩溃
```

**@tmel0211 的用法（154赞那篇）**：
```
说实话，读完这篇"2028的回忆录"，
最让人发毛的倒不是 AI 有多强，
而是它精准地切断了普通人往上走的几条路
```
→ 读者以为「AI 太强了」是结论，实际上被引向了一个更具体的威胁。

**@tmel0211 的另一个用法（139赞那篇）**：
```
恐怕很多人都没想到吧？
量子威胁还没到来之前，
AI 已经敲响了 Crypto 安全的警钟
```

---

### 类型四：数字钩子
**定义**：用让人意外的数字制造冲击，关键是要把比例换算成「你的处境」。

**公式**：`时间极短` + `数字极大/变化极剧烈`

案例一：
```
上线仅40小时的 Polymarket，
成交额竟然就突破了2500万美元
```
案例二：
```
半年前 GPT-5 攻击成功率只有 31.9%，
而现在飙升到了 72.2%
```

案例三：
```
❌ ClawHub 上 7.1% 的 Skills 存在凭证泄露漏洞
✅ 你装了 14 个 Skills，里面可能有 1 个
   正在把你的 API Key 往外发
```

## 四、Don't：会产生「AI 感」的句式

判断标准只有一个：**这句话像是在跟朋友说话，还是在写一篇文章？**

说话是双向的，有温度，有语气，有点随意。写文章是单向的，想把道理说清楚，结果反而显得端着。

以下几种句式，问题不在于「内容不对」，而在于**读起来像在写给人看，而不是在跟人说话**。

---

**1. 身份前置型**

问题不是「不能提身份」，而是提身份的方式像在对陌生人自我介绍，而不是跟熟人说话。

```
❌ 我是做 Agent 安全的，按理说我应该是最懂……
   然后有一天，我才真正明白……
```

→ 「按理说我应该是最懂」是在向读者证明自己，有解释感。

```
✅ 大家都知道我是做 Agent 安全的，所以我们公司
   上上下下都非常注重 Agent 的安全和权限问题。
   结果没想到，新年第一天我的 OpenClaw 差点把
   全公司的信息泄露了。。。
```

→ 「大家都知道」是拉近距离的口语前缀，说明跟读者是熟的。 → 「差点泄露了。。。」有点自嘲，有点戏谑，是在回忆一件让自己也觉得好笑的事。

---

**2. 礼貌转折型**

问题不是「不能转折」，而是转折的方式像老师揭晓答案，站在上方俯视读者。

```
❌ 他说得对，但没有人告诉你的是……
❌ 这个方向是对的，但他没有提到……
```

→ 「没有人告诉你」是在宣布自己掌握秘密，没有平等的对话感。

```
✅ 我觉得她说得很对，但是有一个细节你不要漏掉，
   她能这么说都因为……
```

→ 「你不要漏掉」是朋友提醒朋友，有对话感。 → 「她能这么说都因为」是好奇心驱动，不是在布道。

---

**3. 结论前置型**

```
❌ 那一刻我才真正明白：Agent 的问题从来不是够不够聪明，
   而是边界在哪里、是谁来定的。
```

→ 「那一刻我才真正明白」读出来就很尴尬——故事还没发生，意义先来了，等于替读者思考，也等于剧透。 → 让读者自己从故事里得出结论，比你告诉他结论更有力。

---

**4. 细节缺席型**

```
❌ 它发了一条我没授权的消息
✅ 它发了一封邮件给我们正在谈的合作方，
   内容里有一句话，是我上周内部会议上说的原话，
   当时只有我们四个人在场
```

→ 没有细节的故事读起来是「模板」，不是「经历」。 → 细节不是装饰，是可信度的来源。

---

## 六、写作前自检清单

发出去之前，对照这四个问题：

1. 我的开头能让读者心里「动」一下吗？
2. 核心钩子在前 3 句内出现了吗？
3. 有没有制造足够的认知张力或好奇心？

一句话原则： 先让读者在那个时刻里，再让他们看到你想让他们看到的东西。
