# 钩子、标题与封面文案优化

**启动前先读 `.compound/session.md`**：
- 拿「初稿」段落作为优化对象；拿「选题」段落的核心观点作为标题素材
- 如果还没有初稿，提示用户"要不要先说『用我的文风写』把初稿生成出来？"

初稿写完后，优化三件事：

1. **钩子（开头）** — 诊断当前开头 → 生成 10-15 条优化方案 → 选 Top 3
2. **封面文案（Cover Text）** — 封面图上的文字，给人眼看的，打情绪（自然流量）
3. **内容标题（Content Title）** — 平台标题栏的文字，给算法+人看的，打关键词（搜索流量）

**核心理念：封面文案和内容标题必须写不一样的。写一样的等于浪费了一次机会。**

> "封面图上的标题，和内容标题，不管怎么样，都有两次机会让人点进来，写一样的文字就少了一次机会"

## Part 1：钩子优化（开头）

**核心信念：写不出好开头，90% 是因为内容本身有问题。** 开头是内容的试用装，如果内容没有价值、没有素材、没有冲击力，再怎么优化开头也没用。

### Phase 1：接收文案

用户可能提供：
- 完整初稿（有开头 + 正文）→ 直接进入 Phase 2
- 只有正文（没开头）→ 跳过诊断，直接进入 Phase 3 生成
- 只有选题/标题（连正文都没有）→ **停止。** 「开头是内容的试用装，你连内容都没有，优化开头没有意义。先把正文写出来。」

### Phase 2：诊断

#### 2.1 内容质量门禁

**在优化开头之前，先诊断内容本身有没有问题。**

| 检查项 | 问题 | 诊断 |
|--------|------|------|
| 正文太短 | 少于 200 字 | **警告。** 「正文太短，可能撑不起一个好开头。建议先丰富正文内容。」 |
| 正文没有价值 | 全是废话、没有干货 | **停止优化。** 「正文没有价值，优化开头也没用。先把内容做扎实。」 |

#### 2.2 素材丰富度检查

从文案中找素材：

**有没有冲击力的数据？**
- 大数字（80 亿、400 栋、13000 条）
- 对比数字（从 0 到 X、1 年内）
- 百分比（99%、10 倍）

**有没有转变故事？**
- 之前 vs 之后
- 反差越大越好

**有没有金句？**
- 能独立成立的观点
- 有记忆点、可传播

**有没有权威背书？**
- 人物（巴菲特、犹太富豪）
- 机构（500 强、知名品牌）

**有没有痛点共鸣？**
- 目标人群的焦虑
- 常见的错误做法

**诊断结果**：
- 如果 5 个维度都没有 → **停止优化。** 「你的内容没有素材，写不出好开头。先回去补充数据、故事、金句、权威或痛点。」
- 如果有 1-2 个 → **可以优化，但效果有限。** 「素材不够丰富，开头的冲击力会受限。建议补充更多素材。」
- 如果有 3 个以上 → **可以优化。** 继续下一步。

#### 2.3 话题范围检查

**检查清单**：
- 不是这个领域的人会看吗？
- 没有这个需求的人会好奇吗？
- 有没有缩窄人群的词？

如果话题太窄 → **警告。** 「你的话题太窄，流量上限低。建议用更普世的切入点包装。」

#### 2.4 诊断当前开头（如果有）

| 诊断维度 | 检查项 | 常见问题 |
|---------|--------|---------|
| **独立性** | 不看标题能理解吗？ | 假设用户看了标题，缺少话题建立 |
| **Hook** | 前 5 秒有抓手吗？ | 平铺直叙，没有数据/金句/反差 |
| **悬念** | 有没有直接给答案？ | 开头就说结论，没有好奇心 |
| **可信度** | 为什么听你讲？ | 没有建立权威/成绩/经验 |
| **口播友好** | 能直接念出来吗？ | 自问自答、书面语、抽象概念 |
| **匹配度** | 开头承诺的，正文能兑现吗？ | 标题问 A，正文讲 B |

**输出诊断报告**：

```
当前开头诊断

开头：[用户的开头]

问题：
- 独立性：[问题描述 / OK]
- Hook：[问题描述 / OK]
- 悬念：[问题描述 / OK]
- 可信度：[问题描述 / OK]
- 口播友好：[问题描述 / OK]
- 匹配度：[问题描述 / OK]

核心问题：[最严重的 1-2 个问题]
```

### Phase 3：生成优化方案

**只有通过 Phase 2 的内容质量门禁，才执行这一步。**

用三种方法生成开头，每种 3-5 条，总共 10-15 条。

#### 方法一：素材提取

从文案中提取现有素材，按优先级排序：

| Hook 类型 | 优先级 | 示例 |
|----------|--------|------|
| 晒结果 + 反转 | 最高 | 去年我发了 1 万多条推文，很多人问我执行力怎么这么强，其实我只是搞清楚了一件事 |
| 数据冲击 | 高 | 80 亿、400 栋、13000 条 |
| 反差/转变 | 高 | 从攒首付到管理 80 亿 |
| 金句 | 高 | 不能持有 10 年，就不要持有 10 分钟 |
| 权威+观点 | 中 | 犹太富豪教我一句话 |
| 痛点+悬念 | 中 | 研究 3 个月，焦虑 3 年，后悔 30 年 |

基于提取的素材，生成 3-5 条开头。

#### 方法二：素材增补

如果文案素材不够，主动增补：

**可以增补的素材**：
- 真实数据或结果（如：「我连线了 300 个人发现…」）
- 具体案例或对比
- 反常识的结论

基于增补的素材，生成 3-5 条开头。明确标注增补了什么。

#### 方法三：悬念制造

把结论改成问题，制造好奇心：

**原则**：
- 不要直接给答案
- 用"为什么"而不是"证明"
- 用"问题"而不是"结论"

**例子**：
- 错：李亚鹏 30 年干黄十几个项目，证明人脉不等于赚钱
- 对：善良的李亚鹏，认识半个娱乐圈，30 年为什么赚不到钱

基于悬念制造，生成 3-5 条开头。

### Phase 4：输出方案 + Top 3

```
优化方案（共 X 条）

方法一：素材提取（X 条）
| # | 开头 | 用了什么素材 |
|---|------|------------|

方法二：素材增补（X 条）
| # | 开头 | 增补了什么 |
|---|------|----------|

方法三：悬念制造（X 条）
| # | 开头 | 制造了什么悬念 |
|---|------|-------------|

Top 3 推荐
推荐 1：[开头] — 理由 + 优势
推荐 2：[开头] — 理由
推荐 3：[开头] — 理由
```

### 开头核心公式

```
好开头 = 话题（讲什么）+ Hook（为什么看）+ 可信度（为什么信你）
```

**例子**：
- 话题：选题和标题的区别
- Hook：我去年涨粉 200 万
- 可信度：靠的就是搞清楚这个
- 完整开头：「我去年涨粉 200 万，靠的就是搞清楚选题和标题的区别」

**三条铁律**：
1. 开头必须独立工作——不能假设用户看了标题/封面
2. 开头要制造悬念——不要直接给答案
3. 开头必须口播友好——能直接念出来，不要自问自答

## Part 2：封面文案与内容标题

### 两种文字，两个目的

| 文字类型 | 目的 | 受众 | 流量类型 | 优化方向 |
|---------|------|------|---------|---------|
| **封面文案（Cover Text）** | 显示在封面图/缩略图上 | 人眼滑动 Feed | 自然流量 | 情绪触发：贪嗔痴 |
| **内容标题（Content Title）** | 平台标题/标题栏 | 搜索算法 + 人 | 搜索流量 | 关键词密度 + 可读性 |

**规则：封面文案和内容标题必须不同。**

### 平台规格速查（2026）

| 平台 | 内容标题字数限制 | 封面文案字数限制 | 封面比例 |
|------|----------------|----------------|---------|
| **小红书** | 20 字 | 2-3 行，6-15 字/行 | 3:4（1242x1660） |
| **抖音** | 15-20 字最佳 | 2-3 行，6-12 字/行 | 9:16（1080x1920） |
| **微信视频号** | 16 字推荐 | 2-3 行，6-12 字/行 | 16:9 或 3:4 |
| **YouTube** | 100 字（70 字可见） | 3-5 词 / ~20 字缩略图 | 16:9（1280x720） |
| **X (Twitter)** | 280 字 / 25K（付费） | Article Card 6 词以内 | 16:9（1200x675） |
| **Instagram** | 2200 字 caption（~55 可见） | Reels 封面 3-5 词 | 9:16（1080x1920） |
| **TikTok** | 2200-4000 字描述 | 封面 2-4 词 | 9:16（1080x1920） |
| **Facebook** | 255 字（50 字最佳） | 视频封面 3-5 词 | 16:9 或 1:1 |

### 平台特性规则

**小红书**：
- 封面就是一切——用户滑的是图，不是标题
- 封面文案：大字、2-3 行、情绪钩子为主。数字和清单转化好（"3个方法", "90%的人不知道"）
- 内容标题：严格 20 字。用「」强调。含关键词利于搜索

**抖音**：
- 封面文案模板："X天后...", "99%的人不知道...", "千万别...", "原来...这么简单"
- 黄/白字配深色背景转化好
- 内容标题前置价值主张

**微信视频号**：
- 受众偏年长和专业——少用标题党，多用价值前置
- 封面文案干净、简约、信任感
- 内容标题控制 16 字内，可用 #话题# 增加发现

**YouTube**：
- 缩略图文字最多 5 词，ALL CAPS（英文）。高对比度。120px 缩略图尺寸下必须可读
- 内容标题前 50 字放关键词。数字 + 利益公式
- 核心洞察：缩略图文字应该补充标题，不是重复标题

**X (Twitter)**：
- 知识好奇 + 反常识观点效果最好
- Article Card 标题 + 副标题要搭配使用
- Thread 的第一句就是 hook，必须独立成立

### 四维武器库在封面/标题的应用

#### 封面文案（打情绪 — 哲学 + 心理学为主）

**胡塞尔-意向性锚点**：封面上必须有具体的「某物」让注意力停住

| 原则 | 差的封面 | 好的封面 | 原因 |
|------|---------|---------|------|
| 具体 > 抽象 | "AI改变生活" | "凌晨3点 AI帮我写完了年终报告" | 具体时间+动作=脑中画面 |
| 数字锚定 | "赚了很多钱" | "第37天 收入破万" | "37天"精确到奇怪=可信 |
| 感官细节 | "工作很累" | "连续加班到眼睛充血" | 身体感觉触发具身认知 |

**封面文案公式**：`[具体时间/数字] + [具体动作/场景] + [意外结果]`

**尼采-权力意志断言**：绝对化断言迫使读者反应——同意（分享）或反对（评论），都是互动

| 强度 | 模式 | 中文示例 | 英文示例 |
|------|------|---------|---------|
| 中 | [群体] + [反常识观点] | "程序员最不该学的就是编程" | "The worst thing a developer can learn is code" |
| 强 | [绝对] + [隐含价值判断] | "2026年还在投简历的人 不是懒 是蠢" | "If you're still sending resumes in 2026, you're clueless" |
| 挑衅 | [挑战身份] | "你的努力 其实是一种逃避" | "Your hard work is actually avoidance" |

**平台强度校准**：
- 小红书/抖音：强-挑衅（算法奖励互动）
- 微信视频号：中-强（受众偏好克制）
- YouTube/X：强（反常识表现好，但避免仇恨诱饵）

**福柯-话语权力**：在封面上创造新概念 = 即时建立智识权威

| 命名技巧 | 模式 | 示例 |
|---------|------|------|
| 对比命名 | [旧身份] → [新身份] | "从打工人到「个人公司」" |
| 数字命名 | [具体数字] + [现象] | "「348个月囚徒」"（= 29 年房贷） |
| 反转命名 | [褒义词] + [贬义] | "「高薪穷人」的陷阱" |
| 隐喻命名 | [跨领域] → [你的领域] | "代码农药" / "Prompt Addiction" |
| 时间命名 | [年份] + [综合症] | "2026综合症" / "Post-AI Syndrome" |

**列维纳斯-镜像效应**：最强封面让读者想「这说的就是我」

| 镜像深度 | 模式 | 示例 |
|---------|------|------|
| 浅层「我认识这种人」 | [常见群体] | "北漂人" / "Remote workers" |
| 中层「这是我身边的人」 | [群体] + [日常场景] | "每天通勤2小时的你" |
| 深层「这就是我」 | [场景] + [不愿承认的心理] | "嘴上说躺平 半夜偷偷刷招聘" |

**封面公式**：`[具体群体] + [日常行为] + [他们不愿承认的真相]`

**斯多葛-焦虑重构**：命名焦虑 + 提供重构的标题，表现优于纯贩卖焦虑

| 步骤 | 封面示例 | 内容标题示例 |
|------|---------|------------|
| 只命名焦虑 | "35岁 被裁了" | 太暗，没有出路 |
| 命名 + 重构 | "35岁被裁 反而是最好的事" | "35岁被裁后我才发现：打工才是最大的风险" |
| 命名 + 重构 + 结果 | "35岁被裁 我反而年入百万" | "35岁被裁程序员的逆袭路线图：从失业到自由职业月入10万" |

#### 内容标题（打搜索 — 传播学 + 社会学为主）

**黄金对比句式**：

| 句式 | 公式 | 封面示例 | 内容标题示例 |
|------|------|---------|------------|
| **A but B** | [常识], but [反常识] | "努力没用的" | "努力不是成功的关键，这个被忽视的习惯才是" |
| **Not A, it's B** | 不是[表面], 是[深层] | "不是没钱 是没认知" | "穷的本质不是缺钱，而是缺少这5种认知" |
| **From A to B** | [起点] → [变化] | "从月薪3千到3万" | "月薪从3千到3万：一个普通人的3年转型全记录" |
| **X can Y, but Z** | [事物]能[做到], 但[更大的事] | "AI能写代码" | "AI能写代码，但只有懂这个的人才能驾驭AI" |
| **时间反转** | [过去]以为 X, [现在]知道 Y | "3年前我以为..." | "3年前我以为学历最重要，现在我发现这件事价值10倍" |
| **问题颠覆** | [预期问题] → [意外答案] | "为什么越努力越穷？" | "为什么越努力越穷？因为你努力的方向从一开始就错了" |

**社交货币**——读者转发后获得什么形象：

| 货币类型 | 转发者形象 | 标题策略 | 示例 |
|---------|----------|---------|------|
| **洞察货币** | 有见识、看得透 | 深度分析、反常识 | "AI不会取代你 但会用AI的人会" |
| **态度货币** | 有立场、有原则 | 明确站队、犀利批判 | "拒绝996 不是躺平 是清醒" |
| **资讯货币** | 消息灵通 | 独家信息、趋势预测 | "内部消息：X公司即将..." |
| **实用货币** | 乐于助人 | 干货教程、工具推荐 | "这个免费工具 替我省了10万" |
| **情感货币** | 有温度、有共情 | 暖心故事、共同经历 | "致每一个在深夜崩溃过的成年人" |
| **身份货币** | 属于这个群体 | 圈层信号、部落标记 | "只有自由职业者才懂的痛" |

**每组候选方案必须说清楚提供的是哪种社交货币。**

### 12 种情绪触发器

封面文案的情绪选择速查：

| 分类 | 情绪 | 唤醒度 | 传播力 | 封面模式（中文） | 封面模式（英文） |
|------|------|--------|--------|----------------|----------------|
| **贪** | 欲望 | 高 | 强 | "月入X万的秘密", "0成本..." | "How I made $X in...", "$0 to $X" |
| **贪** | FOMO | 高 | 极强 | "再不学就晚了", "最后X天" | "Before it's too late", "You're already behind" |
| **贪** | 好奇 | 高 | 强 | "原来...这么简单", "很少人知道" | "The secret behind...", "What nobody tells you" |
| **嗔** | 愤怒 | 极高 | 极强 | "凭什么？", "太离谱了" | "This is insane", "How is this legal?" |
| **嗔** | 不公 | 高 | 强 | "被骗了X年", "真相是..." | "You've been lied to", "The truth they hide" |
| **嗔** | 恐惧 | 高 | 强 | "千万别...", "X岁还不懂就完了" | "Stop doing this NOW", "The #1 mistake" |
| **痴** | 归属 | 中 | 强 | "只有X的人才懂", "同类人" | "If you're a..., this is for you" |
| **痴** | 优越 | 中 | 中 | "高手都在用的...", "聪明人的选择" | "What top 1% know", "Elite habits" |
| **痴** | 怀旧 | 中 | 中 | "小时候...", "那些年..." | "Remember when...", "We lost this" |
| 其他 | 希望 | 中 | 中 | "从X到X，我只用了...", "逆袭" | "From $0 to $X", "My comeback story" |
| 其他 | 震惊 | 高 | 强 | "万万没想到", "颠覆认知" | "I can't believe...", "Mind-blowing" |
| 其他 | 共鸣 | 中 | 强 | "你是不是也...", "每次...的时候" | "If you've ever felt...", "We all do this" |

**病毒组合公式**：高唤醒 + 社会属性 = 最大传播
- 最佳组合：`愤怒 + 共鸣`、`恐惧 + 希望`、`好奇 + 震惊`、`FOMO + 欲望`
- 封面文案应该叠加 2 种情绪（主 + 辅）

### 输出格式

每个目标平台生成 **3 组**（封面文案 + 内容标题），每组必须用不同的武器组合：

```
[平台名]

第 1 组（推荐）
| 类型 | 文字 | 字数 |
|------|------|------|
```

## Part 3：Don't——会产生「AI 感」的句式

判断标准只有一个：**这句话像是在跟朋友说话，还是在写一篇文章？**

说话是双向的，有温度，有语气，有点随意。写文章是单向的，想把道理说清楚，结果反而显得端着。

---

**1. 身份前置型**

问题不是「不能提身份」，而是提身份的方式像在对陌生人自我介绍，而不是跟熟人说话。

```
错：我是做 Agent 安全的，按理说我应该是最懂……
   然后有一天，我才真正明白……
```

→ 「按理说我应该是最懂」是在向读者证明自己，有解释感。

```
对：大家都知道我是做 Agent 安全的，所以我们公司
   上上下下都非常注重 Agent 的安全和权限问题。
   结果没想到，新年第一天我的 OpenClaw 差点把
   全公司的信息泄露了。。。
```

→ 「大家都知道」是拉近距离的口语前缀，说明跟读者是熟的。→ 「差点泄露了。。。」有点自嘲，有点戏谑，是在回忆一件让自己也觉得好笑的事。

---

**2. 礼貌转折型**

问题不是「不能转折」，而是转折的方式像老师揭晓答案，站在上方俯视读者。

```
错：他说得对，但没有人告诉你的是……
错：这个方向是对的，但他没有提到……
```

→ 「没有人告诉你」是在宣布自己掌握秘密，没有平等的对话感。

```
对：我觉得她说得很对，但是有一个细节你不要漏掉，
   她能这么说都因为……
```

→ 「你不要漏掉」是朋友提醒朋友，有对话感。→ 「她能这么说都因为」是好奇心驱动，不是在布道。

---

**3. 结论前置型**

```
错：那一刻我才真正明白：Agent 的问题从来不是够不够聪明，
   而是边界在哪里、是谁来定的。
```

→ 「那一刻我才真正明白」读出来就很尴尬——故事还没发生，意义先来了，等于替读者思考，也等于剧透。→ 让读者自己从故事里得出结论，比你告诉他结论更有力。

---

**4. 细节缺席型**

```
错：它发了一条我没授权的消息
对：它发了一封邮件给我们正在谈的合作方，
   内容里有一句话，是我上周内部会议上说的原话，
   当时只有我们四个人在场
```

→ 没有细节的故事读起来是「模板」，不是「经历」。→ 细节不是装饰，是可信度的来源。

## 案例库

### 典型案例

**案例 1：晒结果 + 反转（最高优先级）**
> 去年我发了（1 万多条）推文，同时运营 7 个平台——很多人问我执行力怎么这么强，其实我只是搞清楚了一件事。
- 建立可信度（1 万多条推文）+ 制造好奇（搞清楚了一件事）+ 扩大话题范围

**案例 2：制造悬念，不给答案**
> 善良的李亚鹏，认识半个娱乐圈，30 年为什么赚不到钱？
- 用"为什么"而不是"证明"，留悬念让人想看下去。

**案例 3：开头必须独立工作**
> 一个人赚不到钱的核心原因就是（你上班上多了）。
- 不假设用户看了标题，开头包含完整话题信息。

### 反面案例

**反面 1：假设用户看了标题**
> 标题：一个人赚不到钱的核心原因
> 开头（错误）：（你上班上多了）← 缺少话题建立

**反面 2：开头直接给答案**
> 李亚鹏 30 年干黄十几个项目，证明人脉不等于赚钱。

**反面 3：自问自答，书面语**
> 你以为你在自律？其实你只是在逃避执行。

## 发布前卫生检查（按 CLAUDE.md 的【决策点 3】走）

标题/钩子定稿后，**不要直接问"要不要发布"**。先对全文做一次发布前卫生检查，扫描以下 4 类问题：

1. **空心平行句**：连续出现 3 句以上"不是 X，而是 Y"；排比结构堆砌没有递进；"XXX 是 A，XXX 是 B，XXX 是 C"式的堆叠
2. **干货陷阱**：句式上过度贩卖价值感（"这一点特别重要""一定要记住""99% 的人不知道"），正文里有这类短语超过 2 次要标
3. **开头 3 秒**：第一句能不能抓住一个不认识作者的人？如果第一句是铺垫/背景/概念，视为未过关
4. **承诺-内容一致**：标题/钩子给的承诺，正文是否真的兑现了？标题说 A，正文讲 B，视为未过关

**输出格式**（按 ask-format）：

通过 → 一句话说"都过了，可以发布"+ 引导"发布后说『帮我做发布复盘』"。

未通过 → 列出具体问题点 + 给出选项：
- A) 我按卫生检查的反馈改一版给你看
- B) 我列出每一处问题，你自己改
- C) 先发出去再说，问题留给下次迭代

**总编辑原则**：检出问题就要说，不要因为"这是用户的内容"而沉默放过。用户请你做总编辑，不是做抄写员。
